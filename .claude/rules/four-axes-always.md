# Four axes, always (always active)

Optimise these **simultaneously**, never trade one for another silently:

1. **Speed** — don't over-engineer; ship the smallest correct change.
2. **Accuracy** — it must actually do the ONE thing it claims, correctly.
3. **Quality** — readable, named, within the hard limits, no red flags.
4. **No-guessing** — claims backed by observed evidence ([no-guessing-evidence-only.md](no-guessing-evidence-only.md)).

If a change improves one axis at the cost of another, call out the trade-off
explicitly and get agreement — do not bury it.
