# Changes log

One markdown file per change. Hard cap: **300 lines per file**. If a change needs more, split it.

## Naming

`NNN-short-kebab-slug.md` — zero-padded sequence number, then a 2–5 word summary.

Example: `004-init-claude-md.md`.

## File template

```markdown
# <Title>

**Date:** YYYY-MM-DD
**Type:** fix | feature | refactor | chore | docs

## Problem
<1–3 sentences on what was broken / missing / requested>

## Solution
<What changed, in plain language. Why this approach over alternatives.>

## Files changed
- `path/to/file.ts` — what changed
- ...

## Verification
<How to confirm it works. Commands, what to look at on device, etc.>

## Notes
<Optional. Gotchas, follow-ups, things that surprised us.>
```
