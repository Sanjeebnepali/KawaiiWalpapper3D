# Restore the animated splash + fix its timing so the magnifying glass is seen

**Date:** 2026-05-29
**Type:** fix (supersedes 181 + 182)

## Problem

The owner wanted the **original animated splash** — the loading animation where
a magnifying glass scans across three hearts — but had never actually seen it:

- Changes 181/182 had removed the animated splash entirely (static app icon).
- And even before that (change 180, when the animated splash was present) it
  **dismissed too early to see**: `AnimatedSplash` exited at `MIN_MS = 2000 ms`,
  but the magnifying glass only drops in at ~1100 ms (`ENTER_DELAY 600` +
  `ENTER_MS 500` in `useMagnifyAnimation`) and one full
  left→centre→right→centre sweep takes ~4000 ms. So the splash dismissed before
  the glass left the first heart — it read as a blank purple screen.

## Solution

**Restored the change-180 splash architecture** (the animated splash blended into
a seamless brand-color native splash), via `git checkout 9bfc8b0 --` of the
splash files, then fixed the timing:

- `app/_layout.tsx` — renders `<AnimatedSplash>` again (reverts the 181 removal
  and the 182 `preventAutoHideAsync`/hold logic; `AnimatedSplash` owns the splash
  lifecycle itself).
- `app.json` — native splash back to `backgroundColor #b78fc8` + the transparent
  icon, so the native phase is a solid brand color (no Expo logo, no app icon)
  that hands off invisibly to the animated poster.
- `components/splash/AnimatedSplash.tsx` + `LoadingHearts.tsx` — restored.
- **Timing fix** in `AnimatedSplash.tsx`: `MIN_MS 2000 → 4200`, `MAX_MS
  5000 → 6500`, so a full magnifying-glass sweep across all three hearts plays
  out before the exit spin. This is the actual fix for "I've never seen the
  animation."

Net launch sequence: native `#b78fc8` (no logo) → KawaiiGirl poster fades in →
magnifying glass drops in and scans the three hearts (~4.2 s) → glass spins 360°
as it exits → app.

The animation logic itself (`useMagnifyAnimation` continuous scan +
`useHeartAnimation` lock-step highlight) was already correct — it was never
broken, just cut off by the short `MIN_MS`.
