# Replace the Expo default splash logo with the brand color

**Date:** 2026-05-29
**Type:** fix / config

## Problem

On cold start the app briefly showed the **Expo default splash** — the faint
concentric-circles-on-grid placeholder logo on a white background — before the
JS layer mounted. Cause: `app.json` never configured the `expo-splash-screen`
plugin, so `expo prebuild` generated the stock Expo splash (default
`splashscreen_logo.png` + `#FFFFFF`). The app's real branded splash
(`components/splash/AnimatedSplash.tsx`, which renders the full-screen
`assets/splash-kawaii.png` KawaiiGirl poster with animated hearts) only appears
*after* React mounts and calls `SplashScreen.hideAsync()` — so the Expo
placeholder flashed first. Not acceptable for a production launch.

## Solution

Configure the native splash to render as a **solid brand color with no logo**,
matching the top color of the AnimatedSplash gradient (`GRAD[0] = #b78fc8`) so
the native phase hands off invisibly to the animated poster — one continuous
branded splash, no Expo mark, no color jump. (Chosen by the owner over an
"icon-on-dark" alternative.)

Added the `expo-splash-screen` config plugin to `app.json`:

```json
["expo-splash-screen", {
  "backgroundColor": "#b78fc8",
  "image": "./assets/splash-transparent.png",
  "imageWidth": 200
}]
```

**Why a transparent image instead of just a background color:** on Android 12+
the splash uses the OS `windowSplashScreenAnimatedIcon` API, and the plugin
always wires `styles.xml → @drawable/splashscreen_logo`. Configuring
`backgroundColor` alone left that drawable reference dangling (build break / fall
back to Expo's bundled logo). Pointing `image` at a fully transparent PNG makes
the plugin generate an icon that composites to the background color — i.e.
invisible — so the native splash is a clean solid `#b78fc8` with nothing drawn on
it. Created `assets/splash-transparent.png` (512×512 transparent RGBA, via the
project's bundled `jimp-compact`; a 1×1 source made jimp's resizer throw).

The `android/` project was regenerated with `npx expo prebuild --platform
android`. `android/` is gitignored (prebuild-managed output), so the committed
source of truth is `app.json` + the transparent asset; a fresh clone regenerates
the same splash on build.

The branded poster + animated hearts (`AnimatedSplash`) are unchanged — this only
fixes the native pre-JS phase that preceded them.

## Files changed

- `app.json` — added the `expo-splash-screen` plugin (brand `backgroundColor` + transparent splash image).
- `assets/splash-transparent.png` — **new** 512×512 transparent RGBA PNG (invisible splash icon).
- `android/**` — regenerated by prebuild (gitignored, not committed): `colors.xml` `splashscreen_background = #b78fc8`, `drawable-*/splashscreen_logo.png` regenerated from the transparent source.

## Verification

- `android/app/src/main/res/values/colors.xml` → `splashscreen_background` is
  `#b78fc8` (was `#FFFFFF`).
- The regenerated `drawable-xxxhdpi/splashscreen_logo.png` renders as solid
  `#b78fc8` (the transparent source composited onto the brand color) — **no Expo
  circles**; `styles.xml`'s `windowSplashScreenAnimatedIcon` reference resolves.

  4m 31s** — the transparent-icon drawable resolved (no missing-resource error),
  `app-release.apk` installed.

## Notes

- **Native rebuild required** — splash lives in native resources; a JS reload
  won't pick it up.
- Source of truth is `app.json`; never hand-edit `android/` (regenerated).
- iOS: same config applies on the next iOS prebuild/build (storyboard generated
  with the same `#b78fc8` background) — not built here (Windows env).
