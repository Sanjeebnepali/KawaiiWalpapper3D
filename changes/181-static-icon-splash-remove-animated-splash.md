# Static icon splash — remove the animated splash

**Date:** 2026-05-29
**Type:** change (follow-up to 180)

## Problem

After change 180 made the native cold-start splash a solid brand color
(`#b78fc8`, no logo) that handed off to the JS `AnimatedSplash` (the full-screen
KawaiiGirl poster + animated hearts), the owner — seeing it on the device —
decided that wasn't what they wanted. The request: **show the app's splash icon
directly on launch, with no animated splash and no plain-color screen.** A simple,
static branded splash.

## Solution

**Native splash now shows the app icon.** Pointed the `expo-splash-screen`
plugin `image` at `./assets/app-icon.png` on `backgroundColor: "#000000"` — which
already matches `android.adaptiveIcon.backgroundColor`, so the rounded icon (it
has near-black corners) sits cleanly with no hard edges. The OS shows this icon
on app load and hides it once the first React frame is ready.

**Removed the JS animated splash entirely.** `app/_layout.tsx` no longer renders
`AnimatedSplash`; dropped the `splashVisible` state (and the now-unused `useState`
import). With nothing calling `SplashScreen.preventAutoHideAsync()` anymore, the
native splash uses Expo's default auto-hide — so launch is just: native icon
splash → app, no animation, no extra blank phase.

Deleted the now-dead files:
- `components/splash/AnimatedSplash.tsx` (only consumer of the poster + hearts)
- `components/splash/LoadingHearts.tsx` (only used by AnimatedSplash)
- `assets/splash-kawaii.png` (the poster — only used by AnimatedSplash)
- `assets/splash-transparent.png` (the invisible icon added in 180, now superseded)

`android/` was regenerated with `expo prebuild` (gitignored output; source of
truth is `app.json` + `assets/app-icon.png`).

## Files changed

- `app.json` — splash `image` → `./assets/app-icon.png`, `backgroundColor` → `#000000`.
- `app/_layout.tsx` — removed `AnimatedSplash` import + render + `splashVisible` state + unused `useState` import.
- **Deleted:** `components/splash/AnimatedSplash.tsx`, `components/splash/LoadingHearts.tsx`, `assets/splash-kawaii.png`, `assets/splash-transparent.png`.
- `android/**` — regenerated by prebuild (gitignored, not committed): `splashscreen_background = #000000`, `drawable-*/splashscreen_logo.png` regenerated from `app-icon.png`.

## Verification

- `npx tsc --noEmit` → **exit 0** (removing `AnimatedSplash`/`useState` introduced no errors).
- `npx expo prebuild --platform android` → **Finished prebuild**; `colors.xml`
  `splashscreen_background = #000000`; the regenerated
  `drawable-xxxhdpi/splashscreen_logo.png` shows the kawaii app icon (verified by
  eye), not the Expo placeholder.
- Release APK rebuilt + installed on device (V2231): **BUILD SUCCESSFUL in
  3m 14s**, `app-release.apk` installed.
