# App icons now actually apply (prebuild) + remove broken splash plugin

**Date:** 2026-05-24
**Type:** fix

## Problem
After changes 112–114 the launcher icon and notification icon still showed as the default Android robot glyph on device. Evidence: `android/app/src/main/res/mipmap-*/ic_launcher.webp` was dated **May 14** (untouched), and there was no adaptive-icon XML — i.e. the `app.json` icon config had never been applied to the native project.

## Root cause
`npx expo run:android` only runs `expo prebuild` when the `android/` directory is **missing**. Since `android/` already existed (continuous-native-generation, gitignored), every build just recompiled the stale May-14 resources. So all `app.json` / native-asset changes (launcher icon 113, notification icon 114, native splash 112) never reached the native project — only the JS changes took effect.

A second, latent bug surfaced once prebuild finally ran: the `expo-splash-screen` plugin added in change 112 had a `backgroundColor` + `resizeMode` but **no `image`**, which left a dangling `@drawable/splashscreen_logo` reference and broke `processReleaseResources` ("resource not found"). Since prebuild had never run, this plugin had never actually taken effect anyway.

## Solution
1. Ran `npx expo prebuild --clean --platform android` to regenerate the native project from `app.json`. Verified: `mipmap-*/ic_launcher.webp` regenerated as the logo (71 KB vs the 9 KB default), `ic_launcher.xml` adaptive icon now references `ic_launcher_foreground` (the logo), and `notification_icon.png` (the heart) generated in every density.
2. Removed the incomplete `expo-splash-screen` plugin block from `app.json` (it was breaking resource linking and never worked). The default splash logo is restored by the clean prebuild, so linking passes. The JS animated splash (`components/splash/AnimatedSplash.tsx`) is unaffected — it uses the `expo-splash-screen` *module* API, not the plugin config.
3. Full clean release rebuild (`gradlew assembleRelease`) + installed on both devices.

## Files changed
- `app.json` — removed the `expo-splash-screen` plugin block (the icon config from 113/114 stays).
- (`android/` is gitignored and regenerated by prebuild — not committed.)

## Verification
- Native resources verified as the logo + heart before building (see root-cause notes).
- `gradlew assembleRelease` → `BUILD SUCCESSFUL`; installed `Success` on Vivo V2231 + Samsung Galaxy A14.
- On device: launcher + notification icons show the brand assets (clear the launcher icon cache / reboot if Android still shows a cached old icon).

## Notes
- **Process lesson:** any `app.json` / icon / splash / native-config change requires `npx expo prebuild` BEFORE the build — a plain `expo run:android` will not pick it up when `android/` already exists.
- A branded **native** splash (the brief pre-JS frame) is a possible follow-up: re-add the `expo-splash-screen` plugin *with* a proper `image` so it doesn't dangle. Skipped here to keep the build green and unblock the icons.
